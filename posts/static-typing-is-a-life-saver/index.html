The time I realize that static typing is a life-saver.

Written by Zill_Laiss on 2026-07-23
codinggdscript
بسم الله

It's saved so much of my time without me realizing. Dynamic typing is only good for a while.

Some time earlier, I've been using gdscript and it has both static and dynamic typing. I use static typing of course, but some places dynamic typing unavoidable, like in argument passing (I thought this only apply to std libraries but turns out this apply to every functions).

Anyway, a bug happened and I could not reason why it happened. The logic was correct, and nothing particularly stand out. You know what caused it? Turned out I passed a boolean to an integer argument, then gdscript just decided to convert that boolean into an integer (so either 1 or 0). 

func foo() -> void:
	something(true)

# this is a staticly typed function, 
# or at least that's what I thought...
func something(bar: int) -> void:
	print(bar) # prints 1

No warning, no error, nothing. Just causing weird behavior. Took me a long time since this is impossible to happen in strongly typed languages...

Alhamdu lillāh for static typing.
